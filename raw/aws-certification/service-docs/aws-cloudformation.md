# AWS CloudFormation
> Source: https://docs.aws.amazon.com/whitepapers/latest/aws-overview/management-governance.html
> Collected: 2026-04-09
> Published: Unknown

AWS CloudFormation gives developers and systems
   administrators an easy way to create and manage a collection of related AWS resources,
   provisioning and updating them in an orderly and predictable fashion.

   You can use the AWS CloudFormation sample templates or create your own templates to describe your AWS resources, and any
   associated dependencies or runtime parameters, required to run your application. You don't need
   to figure out the order for provisioning AWS services or the subtleties of making those
   dependencies work. CloudFormation takes care of this for you. After the AWS resources are deployed,
   you can modify and update them in a controlled and predictable way, in effect applying version
   control to your AWS infrastructure the same way you do with your software. You can also
   visualize your templates as diagrams and edit them using a drag-and-drop interface with AWS Infrastructure Composer.
